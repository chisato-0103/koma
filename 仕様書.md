# 仕様書

## システム概要

- システム名：未定
- 目的：大学生活で授業を管理するためのローカルアプリ
- 対象ユーザ：大学生
- 解決したい問題：授業ごとにMoodleやTeamsを横断するのがめんどくさいから一括して管理したい

## 機能一覧

- 機能1
  - 内容：時間割管理（月〜金、1〜6限）
- 機能2
  - 内容：授業とそのMoodle・シラバスURLをリンクさせる
- 機能3
  - 内容：学期（前期・後期など）の切り替え
- 機能4（予定）
  - 必要に応じて追加予定

## 画面設計

- 初回起動時：セットアップ画面（各限の時間を設定 → 最初の学期を作成）
- メイン画面：時間割グリッド（月〜金 × 1〜6限）
- 授業モーダル：授業名・教室・MoodleURL・シラバスURL の入力・コマ数（1/2コマ）選択
- 学期管理モーダル：学期の作成・切替・削除
- ノートページ：セッションバー（第1回・第2回…）＋ 編集／分割／プレビューモード

## データ設計

SQLite（ローカルファイル `userData/timetable.db`）

| テーブル | 概要 |
|---------|------|
| semesters | 学期（id, name, is_active） |
| period_settings | 各限の時間（period_number, start_time, end_time） |
| courses | 授業（id, semester_id, name, day_of_week, period, classroom, moodle_url, syllabus_url, teams_url, note, note_path, present_count, absent_count, period_count） |
| note_sessions | 授業ごとのノートセッション（id, course_id, session_number, note_path, created_at） |

アプリ設定（JSONファイル `userData/settings.json`）

| キー | 概要 |
| --- | --- |
| notes_dir | ノートの保存先ディレクトリ（null の場合は `userData/notes`） |
| portal_url | 大学ポータルのURL（空の場合はボタン非表示） |

## 処理の流れ

1. ユーザがアプリ起動
2. 初回：セットアップ画面で各限の時間・最初の学期を設定
3. メイン画面で時間割を表示
4. セルをクリックして授業を追加・編集
5. 授業詳細からMoodle・シラバスにアクセス（ブラウザで開く）
6. 学期を切り替えることで別学期の時間割を管理

## 技術スタック

| 項目 | 選択 |
|------|------|
| デスクトップフレームワーク | Electron 33 |
| フロントエンド | React 18 |
| ビルドツール | electron-vite |
| バックエンド | Node.js (Electron main process) |
| データベース | SQLite (better-sqlite3 v12) |

## 制約・ルール

- ローカルアプリ（サーバー不要）
- URLは手動入力
- 機能は随時追加予定

## 改善案

- [対応済み] ノートを取れるようにしたい
  - 仕様：授業モーダルにノート入力欄（複数行）を追加し、授業ごとに保存
  - データ：`courses.note`（TEXT）
- [対応済み] 出席数欠席数のカウントをしたい
  - 仕様：授業モーダルで出席・欠席をそれぞれ増減/直接入力可能
  - 表示：時間割セルに `出x / 欠y` を表示
  - データ：`courses.present_count` / `courses.absent_count`（INTEGER）
- [対応済み] PCで起動した時にウィンドウの移動ができない問題
  - 対応内容：ヘッダーをドラッグ領域に指定し、ボタン・セレクトなどの操作要素のみ no-drag 指定
  - 備考：macOS では `hiddenInset` タイトルバー利用時でも移動可能
- [対応済み] 時間割のセルが不均一で気持ち悪い問題
  - 対応内容：時間割テーブルを固定レイアウト化（`table-layout: fixed`）し、左列幅を固定
- [対応済み] ノートをマークダウン対応にしてほしい
  - 対応内容：授業モーダルにMarkdownプレビューを追加（見出し・箇条書き・リンク・コードなど）
- [対応済み] ノートの保存先を最初から.mdファイルにしてほしい
  - 対応内容：`userData/notes` 配下に授業ごとの `.md` ファイルとして保存し、DBは `note_path` で参照
- [対応済み] ノート用のmdファイルの保存場所を設定する場所が欲しい
  - 対応内容：⚙ボタン → 「その他の設定」タブにノート保存先の変更UIを追加（フォルダ選択ダイアログ・リセット・Finderで開く）
  - 設定は `userData/settings.json` に保存。変更後は新規保存から反映（既存ファイルは移動されない）
- [対応済み] どこかに大学のポータルサイトのl-camをリンクさせたい
  - 対応内容：⚙ボタン → 「その他の設定」タブにポータルURLを登録する欄を追加。URL設定後はヘッダーに「ポータル」ボタンが表示され、クリックでブラウザが開く
- [対応済み] UI/UXの見直し
  - 対応内容①：ノートを授業モーダル内のテキストエリアから専用ノートページへの画面遷移方式に変更。左上に「← 時間割」ボタン、編集/分割/プレビュートグル、自動保存（1.5秒）を備えたNotion風UI
  - 対応内容②：授業セルへのホバーで「出席 +1」「欠席 +1」ボタンを表示し、モーダルを開かずにカウント可能に
- [対応済み] moodle以外にteamsで管理する教科もあるからそれにも対応させたい
  - 対応内容：授業モーダルに Teams URL 入力欄を追加（シラバス URL の下）。URLを入力すると「開く」ボタンが表示されデフォルトブラウザで開く
  - データ：`courses.teams_url`（TEXT）
- [対応済み] moodleで管理してる教科とteamsを使用してる教科があって使ってない方が表示されてるのがちょっと気持ち悪いかも
  - 対応内容：URL欄（Moodle・Teams・シラバス）はデフォルト非表示にし、値が登録済みの場合のみ表示。未登録の欄は「+ Moodle」「+ Teams」「+ シラバス」ボタンをクリックすることで展開できる
- [対応済み] ノートのマークダウンが完璧じゃないから完璧にしたい
  - 対応内容：カスタムパーサーを拡張し、テーブル・打消し線（`~~text~~`）・タスクリスト（`- [ ] / - [x]`）・水平線（`---`）・ネストリスト（インデントで自動検出）を追加
- [対応済み] ノートを書くときtabでインデントできるようにしたい
  - 対応内容：ノートページのテキストエリアに `onKeyDown` ハンドラを追加。Tab → 2スペース挿入、Shift+Tab → 先頭2スペース削除。カーソル位置は `useLayoutEffect` で正確に復元
- [対応済み] ノートを回数ごとにファイルを分けたいからそれに対応させたい
  - 対応内容：`note_sessions` テーブルを新設し、授業ごとに第1回・第2回…とセッションを管理。ノートページ上部に回数タブバーを設置し「＋」ボタンで新規回を追加、「×」で削除可能。初回開封時は第1回を自動作成
  - ファイルパス：`{notes_dir}/semester_{id}/{day}/p{period}_c{id}_{slug}/session_{n}.md`
- [対応済み] ノートでエディタを左にプレビューを右に設置し、プレビューを見ながら編集できるようにしてほしい
  - 対応内容：ノートページのモードトグルに「分割」ボタンを追加。左半分がテキストエリア、右半分がリアルタイムMarkdownプレビューの2ペインレイアウト
- [対応済み] 1教科で２コマ使う授業に対応させてほしい
  - 対応内容：授業モーダルに「コマ数」トグル（1コマ / 2コマ）を追加。2コマ選択時は時間割グリッドで2行スパンで表示
  - 制約：2限（昼休みまたぎ）と6限（次限なし）は2コマ不可
  - データ：`courses.period_count`（INTEGER DEFAULT 1）
